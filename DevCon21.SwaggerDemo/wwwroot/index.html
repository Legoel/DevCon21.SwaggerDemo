A worthy (yet insufficient) attempt to provide API documentation

  • Use /api/categories to get all categories as a 'flat list'
  • Use /api/categorytree to get all categories as a 'tree view'
  • Use /api/people to get all registered people
  • Use /api/workitems to get all work items

It is insufficient because:

  • Tedious to set up
  • Difficult to keep up to date
  • Does not handle POST, PUT, PATCH, DELETE requests
  • Does not describe the objects exposed by the service
  • Etc.
